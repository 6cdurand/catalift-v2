## Manual verification still needed

### Messages / Notifications — real 2-account round-trip

1. Create two distinct accounts (A = trainer, B = client) and ensure a trainer-client relationship exists.
2. From `/messages` as **A**, start a conversation and send a message.
3. As **B**, open `/messages`. Confirm:
   - the same message body appears,
   - the unread badge increments,
   - opening the conversation and reading marks it seen (`markConversationSeen`),
   - the badge clears.
4. Reply from **B** and confirm **A** receives it.
5. Send the same message twice (simulate a duplicate realtime INSERT); confirm the UI dedupes by `message.id`.

### PBs — finish→surface e2e

1. Complete a workout with a set that beats an existing PB for `bench-press`.
2. On the summary screen, confirm the PB/summary badge shows the correct canonical e1RM.
3. Navigate to `/pbs`; confirm the list shows the same exercise, weight, reps, and e1RM.
4. Repeat with a weaker set; confirm the `/pbs` list does **not** regress to the lower value.
