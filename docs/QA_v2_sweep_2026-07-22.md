## New bugs / blockers found

### 1. Trainer client-add/invite flow is not wired on staging

**Severity:** BLOCKS C, D, and F.

**Repro steps:**
1. Log in as a trainer.
2. Go to `/clients` and click `Add Client`.
3. Choose either `Create New` or `Link Existing`.
4. Both dialogs display an amber banner: `Adding clients is coming soon — it ships with the trainer-invite update.`
5. In the `Link Existing` dialog the `Link Client` button is disabled/greyed and the text `Linking an existing account is coming soon.` is shown.
6. In `/workout/builder` the `Assign to Client` section shows `No clients found`.

**Impact:** A trainer cannot create or link a client, so `clients/[id]`, one-off workout save, program assignment, and the full CADENCE critical path cannot be exercised end-to-end on the current staging build.

**Regression vs v1 parity:** Likely regression if v1 supported trainer roster management and program assignment to clients. Needs product confirmation.

### 2. `/builder` route advertises a broken "coming soon" builder while `/workout/builder` works

**Severity:** UX / navigation bug.

**Repro steps:**
1. Log in as a trainer.
2. On `/today` click `Open Workout Builder` (or tap the bottom nav `Builder`).
3. The app navigates to `/builder`, which renders `Standalone workout builder is coming soon`.
4. Manually navigate to `/workout/builder` in the same session.
5. The real one-off builder loads: it accepts blocks, exercises, and a workout name.

**Impact:** Trainers are likely to think the one-off builder is unavailable because the primary CTA points to a placeholder route instead of the working one.

### 3. One-off `Save Workout` requires a client assignment with no fallback

**Severity:** BLOCKS D in combination with bug #1.

**Observations:** In `/workout/builder`, after adding a `Strength` block and an exercise, the `Save Workout` button remains disabled because no client is selected. There is no `Save as template`, `Assign to me`, or `None (template only)` fallback. Even if the client-add flow were fixed, the save semantics should be reviewed to ensure a trainer can save a one-off workout before assigning it.
